Bloody and Horrible Frame Detection

This project involves the automatic detection of violent and bloody frames using OpenCV. The goal is to detect and classify graphic content in real-time, enhancing safety and content moderation for various applications like security footage, movies, and online platforms.

  • Tools Used: OpenCV, Python
  • Dataset: 2000 images
  • Accuracy Achievements:
    • Violent Frames: 82%
    • Bloody Frames: 83%
    • Normal Frames: 90%

This model successfully balances precision and performance in identifying violent and disturbing content, making it ideal for content-sensitive environments.
